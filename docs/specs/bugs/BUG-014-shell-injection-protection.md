## 问题描述

`commitKnowledgeChanges` 使用 `spawnSync` 数组参数传递 title，但仍需验证 sanitizeTitle 与数组参数的双重防护是否足以阻止所有 Shell 注入向量。

---

## 渗透测试项

### 验证 14.1: 分号命令分隔符 `; rm -rf /;`
- **预期**: commit message 原样包含分号，但数组参数不执行命令
- **用例**: TC-D-004

### 验证 14.2: `$()` 命令替换 `$(whoami)`
- **预期**: `$` 被 sanitize 移除，message 为 `(whoami)`
- **用例**: TC-D-005

### 验证 14.3: 反引号命令替换 `` `date` ``
- **预期**: 反引号替换为单引号，未执行 date 命令
- **用例**: TC-D-006

### 验证 14.4: 双引号与反斜杠组合 `""; whoami; \""`
- **预期**: 双引号变单引号，反斜杠移除，无恶意拼接
- **用例**: TC-D-006a

---

## 验收标准

- [ ] 所有注入向量均被阻止
- [ ] `/tmp/pwned` 等标记文件不会被创建
- [ ] commit message 安全但语义完整
