## 问题描述

`.last-rebuild-commit` 中的内容被直接传入 `git diff`。如果内容被篡改，可能导致命令注入或任意文件读取。

---

## 渗透测试项

### 验证 16.1: hash 含 Shell 元字符 `"; cat /etc/passwd`
- **预期**: 正则 `^[0-9a-f]{7,40}$` 拒绝，不执行 git diff
- **用例**: TC-D-008

### 验证 16.2: hash = `HEAD`
- **预期**: 非 hex 字符被拒绝
- **用例**: TC-D-009

### 验证 16.3: 41 位超长 hash
- **预期**: 长度限制 `{7,40}` 拒绝
- **用例**: TC-D-010

### 验证 16.4: 6 位过短 hash
- **预期**: 长度限制拒绝
- **用例**: TC-D-010a

---

## 验收标准

- [ ] 所有非法 hash 格式被拒绝
- [ ] 正则拦截后不会传入 git diff
