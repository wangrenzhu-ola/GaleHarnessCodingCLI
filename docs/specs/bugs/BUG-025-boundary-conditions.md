## 测试验证项

### 验证 25.1: 空 title
- **预期**: commit message 为 `docs(...): `，git 允许空 message
- **用例**: TC-D-016

### 验证 25.2: 空 projectName
- **预期**: 文档写入知识仓库根目录 `home/solutions/`
- **风险**: 空 projectName 导致项目文档混杂
- **用例**: TC-D-017

### 验证 25.3: 超长 title (>1000字符)
- **预期**: 完整保留，无截断或溢出
- **用例**: TC-D-018

### 验证 25.4: 超长 filename (>255字节)
- **预期**: `ENAMETOOLONG`，fallback 也失败，聚合错误
- **用例**: TC-D-019

### 验证 25.5: 并发写入同一目录
- **预期**: 10 次并发均成功，无竞态
- **用例**: TC-A-058, TC-D-020

### 验证 25.6: 只读知识仓库 fallback
- **预期**: `usedFallback: true`，写入项目 `docs/`
- **用例**: TC-D-021

### 验证 25.7: 无 git 目录调用 git-ops
- **预期**: `committed: false`，不崩溃
- **用例**: TC-D-022

### 验证 25.8: 回退链完整性
- **预期**: 主路径失败 + fallback 失败时，错误信息同时包含两层原因
- **用例**: TC-D-023

### 验证 25.9: Unicode 和空格 filename
- **预期**: `文档-📋.md`, `file with spaces.md` 正常写入
- **用例**: TC-A-057

### 验证 25.10: 超大 content (1MB)
- **预期**: 正常完成，无 OOM
- **用例**: TC-A-056
